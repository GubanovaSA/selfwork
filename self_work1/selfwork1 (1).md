## Задание 2: Создание схемы и таблиц

### SQL-код создания структуры (4 таблицы)

```sql
-- ============================================
-- СХЕМА CRM (Управление взаимоотношениями с клиентами)
-- ============================================

-- 1. Таблица companies (компании-клиенты)
CREATE TABLE IF NOT EXISTS companies (
    company_id SERIAL PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    industry VARCHAR(100),
    website VARCHAR(100)
);

-- 2. Таблица contacts (контактные лица)
CREATE TABLE IF NOT EXISTS contacts (
    contact_id SERIAL PRIMARY KEY,
    company_id INT REFERENCES companies(company_id),
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE
);

-- 3. Таблица deals (сделки)
CREATE TABLE IF NOT EXISTS deals (
    deal_id SERIAL PRIMARY KEY,
    company_id INT REFERENCES companies(company_id),
    deal_name VARCHAR(200) NOT NULL,
    stage VARCHAR(50),
    amount NUMERIC(12,2)
);

-- 4. Таблица activities (активности: звонки, email, встречи)
CREATE TABLE IF NOT EXISTS activities (
    activity_id SERIAL PRIMARY KEY,
    contact_id INT REFERENCES contacts(contact_id),
    type VARCHAR(50),
    notes TEXT,
    activity_date DATE DEFAULT CURRENT_DATE
);
```

### Результат

Созданы 4 таблицы в схеме `public`:

| Таблица | Описание | Ключевые поля |
|---------|----------|---------------|
| **companies** | Компании-клиенты | company_id, name, industry, website |
| **contacts** | Контактные лица | contact_id, company_id, first_name, last_name, email |
| **deals** | Сделки | deal_id, company_id, deal_name, stage, amount |
| **activities** | Активности | activity_id, contact_id, type, notes, activity_date |

### Распределение сделок по отраслям

```sql
SELECT 
    c.industry,
    COUNT(DISTINCT c.company_id) AS companies,
    COUNT(d.deal_id) AS deals,
    SUM(d.amount) AS total_amount,
    ROUND(AVG(d.amount), 2) AS avg_amount
FROM companies c
LEFT JOIN deals d ON c.company_id = d.company_id
GROUP BY c.industry
ORDER BY total_amount DESC;
```

##  Задание 5: JOIN запрос (аналитика CRM)

###  JOIN запрос для агрегированных данных

```sql
-- Аналитика эффективности работы с клиентами
SELECT 
    c.name AS компания,
    c.industry AS отрасль,
    COUNT(DISTINCT cont.contact_id) AS количество_контактов,
    COUNT(DISTINCT d.deal_id) AS количество_сделок,
    COUNT(DISTINCT CASE WHEN d.stage = 'Закрытие выигрыш' THEN d.deal_id END) AS выигранные_сделки,
    COUNT(DISTINCT CASE WHEN d.stage = 'Закрытие проигрыш' THEN d.deal_id END) AS проигранные_сделки,
    ROUND(COUNT(DISTINCT CASE WHEN d.stage = 'Закрытие выигрыш' THEN d.deal_id END) * 100.0 / 
          NULLIF(COUNT(DISTINCT d.deal_id), 0), 2) AS процент_выигрыша,
    COUNT(DISTINCT a.activity_id) AS количество_активностей,
    SUM(d.amount) AS общая_сумма_сделок,
    ROUND(AVG(d.amount), 2) AS средний_чек
FROM companies c
LEFT JOIN contacts cont ON c.company_id = cont.company_id
LEFT JOIN deals d ON c.company_id = d.company_id
LEFT JOIN activities a ON cont.contact_id = a.contact_id
GROUP BY c.name, c.industry
ORDER BY выигранные_сделки DESC, общая_сумма_сделок DESC;
```

## Чарт 1: Столбчатая диаграмма "Эффективность работы с клиентами"

### Описание чарта
Столбчатая диаграмма, отображающая для каждой компании количество **выигранных** и **проигранных** сделок. По оси X — компании, по оси Y — количество сделок. Цвета разделяют выигранные (зеленый) и проигранные (красный) сделки.

### Что демонстрирует

| Бизнес-процесс | Как отражается на чарте |
|----------------|------------------------|
| **Воронка продаж** | Показывает успешные и неуспешные завершения сделок |
| **Клиентский портфель** | Видно, какие компании приносят больше успешных сделок |
| **Эффективность менеджеров** | Соотношение выигрыш/проигрыш отражает качество работы |
| **Управление рисками** | Компании с большим количеством проигрышей требуют внимания |

### Аналитика по данным

На основе представленных данных (проценты на оси Y):

| Компания/Отрасль | Выигранные сделки | Проигранные сделки | Соотношение |

| IT | 923% | 436% | 2.1 : 1 |
| Ритейл | 957% | 440% | 2.2 : 1 |

| Строительство | 853% | 750% | 1.1 : 1 |

### Выводы

1. **Наиболее успешные отрасли:**
   - IT, Ритейл, Финансы показывают соотношение выигрыш/проигрыш более 2:1
   - Это стабильные и перспективные направления для бизнеса

2. **Проблемные отрасли:**
   - Производство имеет больше проигрышей (853%) чем выигрышей (734%)

   - Необходим анализ причин проигрышей в этих отраслях
